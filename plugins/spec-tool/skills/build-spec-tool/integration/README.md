# Platform-Specific Integration

This skill generates platform-agnostic `.ai-dev/` knowledge files that work with any AI agent system. The `integration/` directory contains platform-specific examples and guides:

## Available Integrations

- **[Claude](./claude/)** — Uses Claude CLI or Copilot Chat. Recommended for VS Code users.
- **[GPT](./gpt/)** — Uses OpenAI API (gpt-4, gpt-4-turbo). Good for those with OpenAI subscriptions.
- **[Copilot](./copilot/)** — Uses GitHub Copilot. Integrated with VS Code and GitHub.

## How to Use

1. **Generate `.ai-dev/` files** by running the build-spec-tool skill
2. **Choose your platform** from the list above
3. **Follow the integration guide** for your chosen platform
4. **Load the `.ai-dev/` files** as context in your AI agent
5. **Start planning** using the SYSTEM_PROMPT.md methodology

## Adding a New Platform

If you want to add integration examples for another AI platform (e.g., Google Gemini, Anthropic Prompt Caching):

1. Create a new directory: `integration/platform-name/`
2. Add a `README.md` with setup instructions and examples
3. Add any platform-specific scripts or configuration files
4. Submit as a contribution!

## Key Files

All integrations use the same core files generated by build-spec-tool:

- `.ai-dev/SYSTEM_PROMPT.md` — Planning methodology
- `.ai-dev/PLAN.md` — Plan template
- `.ai-dev/ARCHITECTURE.md` — Tech stack and conventions
- `.ai-dev/SECURITY.md` — Security policies
- `.ai-dev/LEGAL.md` — Legal and compliance requirements
- `.agents/plan-review-agent.md` — Plan validator

The integration guides show how to load these files into each platform.
